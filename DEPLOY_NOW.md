## Why Render for Backend?

Vercel serverless has limitations for this backend:
- ❌ 10-second timeout (resume processing takes longer)
- ❌ Ephemeral storage (database resets frequently)
- ❌ Cold starts cause 5-10 second delays

Render provides:
- ✅ No timeout limits
- ✅ Persistent disk storage for SQLite
- ✅ Always-on backend (no cold starts on paid plans)
- ✅ Free tier available

## 🎯 ONE-CLICK DEPLOYMENT TO RENDER

### Option 1: Blueprint Deployment (Easiest)

**Click this link**: 

https://dashboard.render.com/select-repo?type=blueprint

1. Sign in to Render with GitHub
2. Select repository: `victorvengatesh/resume-analyzer`
3. Render auto-detects `render.yaml` and configures everything
4. **Only add**: `GEMINI_API_KEY` (your API key)
5. Click **"Apply"** → Wait 3-5 minutes → Done!

Your backend URL will be: `https://resume-analyzer-backend-xxxx.onrender.com`

### Option 2: Manual Setup

If blueprint doesn't work:

1. Go to https://dashboard.render.com
2. Click **"New +"** → **"Web Service"**
3. Connect GitHub: `victorvengatesh/resume-analyzer`
4. Configure:
   - **Name**: `resume-analyzer-backend`
   - **Root Directory**: `./`
   - **Build Command**: `pip install -r backend/requirements.txt`
   - **Start Command**: `cd backend && uvicorn main:app --host 0.0.0.0 --port $PORT`
   - **Environment**: Python 3.11
   
5. Add environment variables:
   ```
   PYTHON_VERSION=3.11.11
   PYTHONPATH=.
   APP_MODE=production
   ENABLE_AUTH=false
   FRONTEND_URL=https://frontend-mztwp8yvz-victorvengateshs-projects.vercel.app
   DATABASE_URL=sqlite:///data/resume_ai.db
   GEMINI_API_KEY=<your-key>
   SECRET_KEY=<auto-generated-or-use: d47269471e81176b3c2d6f96dab01bceefba4d8b2a962803c427a52bc3fb7aac>
   ```

6. Add **Persistent Disk**:
   - Name: `resume-data`
   - Mount Path: `/data`
   - Size: 1 GB

7. Click **"Create Web Service"**

## 🆚 Vercel Backend vs Render Backend

| Feature | Vercel Serverless | Render Web Service |
|---------|-------------------|-------------------|
| **Timeout** | 10s (free) / 60s (pro) | Unlimited |
| **Storage** | Ephemeral (/tmp) | Persistent disk |
| **Cold Starts** | Yes (5-10s delay) | No (on paid plans) |
| **Database** | Resets frequently | Persists forever |
| **Cost** | Free tier limited | Free tier generous |
| **Best For** | Simple APIs | Full applications |

For this resume analyzer with AI processing and database needs, **Render is the clear winner**.
